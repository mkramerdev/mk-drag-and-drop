# Quickstart

`mk-drag-and-drop` is headless. You provide elements and register the behavior you want.

## Installation

```bash
pnpm add @mk-drag-and-drop/dom
pnpm add @mk-drag-and-drop/react
```

## Refine Drop Behavior

Once a single drop target works, register both containers as drop targets and decide what your app should do after a successful drop.
`mk-drag-and-drop` only tells you that a valid drop happened. Your app owns the behavior: move an element,
set React state, persist data, trigger a command, or do nothing.

This example moves the item between both containers and applies a ghost style to the dragged item while the drag is active.
The targeting constraint accepts only drops directly on a target.
