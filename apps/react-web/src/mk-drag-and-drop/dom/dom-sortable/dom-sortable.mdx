# DOM Sortable

DOM sortable behavior is for reorderable lists, rows, columns, and board cards. Use `createSortable` when the same element should be both a drag source and a drop target.

Use plain `createDraggable` and `createDroppable` when dragging one thing into a target is enough and you do not need reorder preview, before/after placement, or sortable placement data.

A sortable item participates in the same controller scope, groups, targeting, lifecycle callbacks, overlays, modifiers, and remeasurement behavior as the rest of `@mk-drag-and-drop/dom`.

## Minimal Sortable List

```ts
import {
  createDragController,
  createSortable,
  type SortableDropPlacement,
} from "@mk-drag-and-drop/dom";

type Item = {
  id: string;
  label: string;
};

const listElement = document.querySelector<HTMLElement>("#items");

if (!listElement) {
  throw new Error("Missing #items");
}

let items: Item[] = [
  { id: "task-1", label: "Write outline" },
  { id: "task-2", label: "Review API" },
  { id: "task-3", label: "Publish notes" },
];

const controller = createDragController({
  onDrop({ draggableId, sortablePlacement }) {
    if (!sortablePlacement) {
      return;
    }

    items = moveItemToSortablePlacement(
      items,
      draggableId,
      sortablePlacement
    );
    render();
  },
});

function render(): void {
  listElement.replaceChildren(
    ...items.map((item) => {
      const element = document.createElement("div");
      element.className = "row";
      element.textContent = item.label;

      createSortable({
        controller,
        element,
        draggableId: item.id,
        group: "tasks",
      });

      return element;
    })
  );
}

function moveItemToSortablePlacement(
  currentItems: readonly Item[],
  draggableId: string,
  placement: SortableDropPlacement
): Item[] {
  const item = currentItems.find(
    (currentItem) => currentItem.id === draggableId
  );

  if (!item) {
    return [...currentItems];
  }

  const withoutItem = currentItems.filter(
    (currentItem) => currentItem.id !== draggableId
  );
  const insertIndex = getPlacementIndex(
    withoutItem.map((currentItem) => currentItem.id),
    placement
  );

  return [
    ...withoutItem.slice(0, insertIndex),
    item,
    ...withoutItem.slice(insertIndex),
  ];
}

function getPlacementIndex(
  ids: readonly string[],
  placement: SortableDropPlacement
): number {
  if (placement.targetDraggableId !== null && placement.side !== null) {
    const targetIndex = ids.indexOf(placement.targetDraggableId);

    if (targetIndex !== -1) {
      return placement.side === "after" ? targetIndex + 1 : targetIndex;
    }
  }

  if (placement.previousDraggableId !== null) {
    const previousIndex = ids.indexOf(placement.previousDraggableId);

    if (previousIndex !== -1) {
      return previousIndex + 1;
    }
  }

  if (placement.nextDraggableId !== null) {
    const nextIndex = ids.indexOf(placement.nextDraggableId);

    if (nextIndex !== -1) {
      return nextIndex;
    }
  }

  return ids.length;
}

render();
```

The DOM package owns drag mechanics and preview movement. Your app still owns the item array and final render.

## `createSortable` Options

`createSortable(input)` returns `void`.

```ts
type CreateSortableInput = {
  controller: DragController;
  element: HTMLElement;
  draggableId: string;
  group?: string;
  containerId?: string | null;
  axis?: "vertical" | "horizontal";
  placementBoundary?: {
    start?: number;
    end?: number;
  };
};
```

Required options:

- `controller`: controller scope that owns registrations, targeting, lifecycle callbacks, overlays, and remeasurement.
- `element`: DOM element to register as the sortable item.
- `draggableId`: stable id for the sortable item. This is also the sortable item's drop target id.

Optional options:

- `group`: matching group for draggable and drop target behavior. Defaults to `"default"`.
- `containerId`: logical container containing the item. Defaults to `null`.
- `axis`: sortable placement axis. Defaults to `"vertical"`.
- `placementBoundary`: reversal placement ratios. Defaults to `{ start: 0.25, end: 0.75 }`.

A sortable item is both draggable and droppable. It can start a pointer or keyboard drag, and it can become the active drop target for another sortable item in the same group.

## Controller Setup

Lifecycle callbacks are configured on `createDragController`, not on individual sortable items.

```ts
import { createDragController } from "@mk-drag-and-drop/dom";

const controller = createDragController({
  onDragStart({ draggableId, source }) {
    console.log("Started", draggableId, source);
  },

  onDragUpdate({ activeDropTargetId, previousDropTargetId }) {
    if (activeDropTargetId !== previousDropTargetId) {
      console.log("Active target changed", activeDropTargetId);
    }
  },

  onDragEnd({ result, dropTargetId }) {
    console.log("Ended", result, dropTargetId);
  },

  onDrop({ draggableId, dropTargetId, sortablePlacement }) {
    console.log("Dropped", draggableId, dropTargetId, sortablePlacement);
  },
});
```

For sortable ordering, commit durable app state in `onDrop` when `sortablePlacement` is present. `onDragEnd` runs before `onDrop` for a successful drop and does not include `sortablePlacement`.

Use `onDragUpdate` for active styling and lightweight feedback. Avoid committing order on every update.

## Targeting vs Placement

Targeting and sortable placement are related but separate.

### Targeting

Targeting chooses the active drop target from registered targets in the active group. The public signals are `DragUpdateEvent.activeDropTargetId` and `DropEvent.dropTargetId`.

### Placement

Placement decides before/after or container placement relative to the active target. The public signal is `DropEvent.sortablePlacement`.

`dropTargetId` is the active target that won targeting. In a sortable board, that target can be an item id or a container id. Use `sortablePlacement.containerId` as the destination container when sortable placement is present.

### Lifecycle

Lifecycle callbacks let the app observe the drag and commit state through `onDragStart`, `onDragUpdate`, `onDragEnd`, and `onDrop`.

The controller's `targetingAlgorithm` chooses the active target. Sortable placement does not choose the target, and `placementBoundary` is not a targeting threshold.

## Committing Order

Sortable placement is exposed on `DropEvent.sortablePlacement`.

```ts
type SortableDropPlacement = {
  sourceContainerId: string | null;
  containerId: string | null;
  previousDraggableId: string | null;
  nextDraggableId: string | null;
  targetDraggableId: string | null;
  side: "before" | "after" | null;
};
```

Field meanings:

- `sourceContainerId`: dragged item's container at drag start.
- `containerId`: destination container. This can be `null` for sortable sets that do not use containers.
- `previousDraggableId`: nearest sortable item before the final position, when available.
- `nextDraggableId`: nearest sortable item after the final position, when available.
- `targetDraggableId`: active sortable item anchor when the drop lands relative to an item.
- `side`: `"before"` or `"after"` relative to `targetDraggableId`, or `null` for container-only placement.

A practical single-list commit helper can prefer the exact active target anchor first, then fall back to sibling anchors:

```ts
import type { SortableDropPlacement } from "@mk-drag-and-drop/dom";

function reorderIds(
  ids: readonly string[],
  draggableId: string,
  placement: SortableDropPlacement
): string[] {
  const withoutDragged = ids.filter((id) => id !== draggableId);
  let insertIndex = withoutDragged.length;
  let resolvedPlacement = false;

  if (placement.targetDraggableId !== null && placement.side !== null) {
    const targetIndex = withoutDragged.indexOf(placement.targetDraggableId);

    if (targetIndex !== -1) {
      insertIndex = placement.side === "after" ? targetIndex + 1 : targetIndex;
      resolvedPlacement = true;
    }
  }

  if (!resolvedPlacement && placement.previousDraggableId !== null) {
    const previousIndex = withoutDragged.indexOf(
      placement.previousDraggableId
    );

    if (previousIndex !== -1) {
      insertIndex = previousIndex + 1;
      resolvedPlacement = true;
    }
  }

  if (!resolvedPlacement && placement.nextDraggableId !== null) {
    const nextIndex = withoutDragged.indexOf(placement.nextDraggableId);

    if (nextIndex !== -1) {
      insertIndex = nextIndex;
    }
  }

  return [
    ...withoutDragged.slice(0, insertIndex),
    draggableId,
    ...withoutDragged.slice(insertIndex),
  ];
}
```

`sortablePlacement` can be omitted. Treat that as no sortable order change unless your app has another reason to update state.

## Containers And Empty Lists

For one simple list, `containerId` can be omitted. For boards, lanes, columns, or empty lists, register containers with `createDropContainer` and give sortable items a matching `containerId`.

In multi-container sortables, treat `sortablePlacement.containerId` as the destination and `sortablePlacement.sourceContainerId` as the drag-start source. Remove the dragged id from every container first, then insert it into the destination using `targetDraggableId`, `previousDraggableId`, and `nextDraggableId`.

```ts
import {
  createDragController,
  createDropContainer,
  createSortable,
  type SortableDropPlacement,
} from "@mk-drag-and-drop/dom";

type Lane = {
  id: string;
  cardIds: string[];
};

let lanes: Lane[] = [
  { id: "todo", cardIds: ["a", "b"] },
  { id: "done", cardIds: [] },
];

const controller = createDragController({
  onDrop({ draggableId, sortablePlacement }) {
    if (!sortablePlacement) {
      return;
    }

    lanes = moveCard(lanes, draggableId, sortablePlacement);
    renderBoard();
  },
});

function renderLane(lane: Lane): HTMLElement {
  const laneElement = document.createElement("section");

  createDropContainer({
    controller,
    element: laneElement,
    containerId: lane.id,
    group: "cards",
  });

  for (const cardId of lane.cardIds) {
    const cardElement = document.createElement("article");
    cardElement.textContent = cardId;

    createSortable({
      controller,
      element: cardElement,
      draggableId: cardId,
      group: "cards",
      containerId: lane.id,
    });

    laneElement.append(cardElement);
  }

  return laneElement;
}

function moveCard(
  currentLanes: readonly Lane[],
  draggableId: string,
  placement: SortableDropPlacement
): Lane[] {
  if (placement.containerId === null) {
    return [...currentLanes];
  }

  return currentLanes.map((lane) => {
    const idsWithoutCard = lane.cardIds.filter((id) => id !== draggableId);

    if (lane.id !== placement.containerId) {
      return { ...lane, cardIds: idsWithoutCard };
    }

    return {
      ...lane,
      cardIds: reorderIds(idsWithoutCard, draggableId, placement),
    };
  });
}

function renderBoard(): void {
  // App-owned rendering omitted.
}
```

Empty containers can become valid drop regions when registered with `createDropContainer`. A drop into an empty container can produce a placement where `previousDraggableId`, `nextDraggableId`, `targetDraggableId`, and `side` are all `null`; use `containerId` to append the item to that container.

For non-empty destination containers, the active target may be a child sortable item. For broad or empty destination regions, the active target may be the container itself. In both cases, the commit path should read the normalized destination from `sortablePlacement.containerId`.

## Axis

Sortable supports two public axis values:

- `"vertical"`: lists, menus, and stacked rows.
- `"horizontal"`: columns, tabs, and horizontal boards.

The default axis is `"vertical"`.

```ts
createSortable({
  controller,
  element: rowElement,
  draggableId: row.id,
  group: "rows",
  axis: "vertical",
});
```

For horizontal sorting, set `axis: "horizontal"` on the sortable items in that sortable group or container.

```ts
createSortable({
  controller,
  element: columnElement,
  draggableId: column.id,
  group: "columns",
  containerId: "board",
  axis: "horizontal",
});
```

Use one axis consistently for items that are intended to sort together.

## Placement Boundaries

`placementBoundary` tunes when sortable placement switches while the pointer or overlay center reverses direction over the same active target.

```ts
createSortable({
  controller,
  element,
  draggableId: item.id,
  group: "items",
  placementBoundary: {
    start: 0,
    end: 1,
  },
});
```

The option shape is:

```ts
type SortablePlacementBoundary = {
  start?: number;
  end?: number;
};
```

Defaults are:

```ts
{
  start: 0.25,
  end: 0.75,
}
```

The values are ratios of the active target on the sortable axis:

- `start`: used when movement is forward on the sortable axis. Placement can switch after the position crosses the start ratio.
- `end`: used when movement is backward on the sortable axis. Placement can switch before the position crosses the end ratio.

On first entry into a newly active target in the same container, sortable placement remains movement-responsive: forward movement places after, backward movement places before, and no movement falls back to the midpoint. When entering a different DOM container, the initial side is midpoint-based using the current sortable placement position. Pointer-based targeting uses the pointer as that position; rect-based targeting such as `centerToCenter` uses the overlay center. Boundaries are same-target reversal and hysteresis controls; they do not decide the initial cross-container side or delay the first movement after midpoint-based entry.

Compact rows often work well with defaults. Large targets may benefit from wider hysteresis so placement does not flip too easily during small pointer reversals.

## Drag Handles With Sortable

`createSortable` can be combined with `createDragHandle`. Mark the handle element inside the sortable item.

```ts
import {
  createDragHandle,
  createSortable,
} from "@mk-drag-and-drop/dom";

const itemElement = document.createElement("div");
const handleElement = document.createElement("button");

handleElement.type = "button";
handleElement.textContent = "Drag";

itemElement.append(handleElement, document.createTextNode("Item label"));

createSortable({
  controller,
  element: itemElement,
  draggableId: "item-1",
  group: "items",
});

createDragHandle({
  element: handleElement,
});
```

Use handles when the sortable item also contains buttons, links, inputs, or selectable text.

## Remeasurement

Drop targeting uses measured target rectangles. The package measures targets when they are registered and at drag start. During normal sortable preview movement, app code does not need to call remeasurement just to move the preview.

Call `controller.remeasureDropTargets` after app-owned layout changes that should affect targeting during a drag. Common examples are expanding rows, collapsing sections, loading more items, changing item heights, or showing and hiding nested drop targets.

```ts
function toggleExpanded(): void {
  expanded = !expanded;
  render();

  requestAnimationFrame(() => {
    controller.remeasureDropTargets({ group: "items" });
    controller.recomputeActiveDrag();
  });
}
```

Supported public inputs are:

```ts
controller.remeasureDropTargets();
controller.remeasureDropTargets("item-1");
controller.remeasureDropTargets(["item-1", "item-2"]);
controller.remeasureDropTargets({ group: "items" });
```

Use the narrowest input that matches the layout change. Remeasurement updates cached target rectangles; it does not send a drag update by itself. If the active target or sortable preview should update before the next pointer or keyboard movement, call `controller.recomputeActiveDrag()` after remeasurement.

## Cancel And Cleanup Behavior

Sortable preview movement is temporary. During a drag, the package may move the dragged DOM element to show the proposed position. On cancel, no-target release, invalid-target release, active drag reset, and successful drop, the package restores sortable preview state.

Your app does not need to manually undo preview DOM movement. Commit app state from lifecycle callbacks, usually `onDrop`, and render the final state from your data.

`onDragEnd` includes:

```ts
type DragEndResult =
  | "dropped"
  | "no-target"
  | "invalid-target"
  | "canceled";
```

`onDrop` only fires for a valid successful drop. Use `onDragEnd` for cleanup such as clearing active styling. Use `onDrop` for sortable order commits.

Root DOM bindings such as `createSortable` return `void`; they do not expose a per-item disposer. If you remove or replace DOM elements, keep ids stable. Disconnected bindings are pruned on later drag start and remeasurement paths; if elements change during an active drag and that should affect targeting, call `controller.remeasureDropTargets` and then `controller.recomputeActiveDrag`.

## Performance Notes

- Keep sortable ids stable. Changing ids forces the drag-and-drop layer and your app state to treat the item as a different sortable item.
- Keep lifecycle handlers fast. `onDragUpdate` can run frequently, so use it for lightweight visual state, not full data commits.
- Commit durable order in `onDrop`, not on every pointer move. The sortable preview already handles temporary visual movement.
- For large vanilla DOM lists, reuse DOM nodes when practical. Replacing thousands of elements after every small state change also re-registers thousands of sortable bindings.
- Remeasure only after layout changes that should affect targeting, then recompute when the active target should update immediately.
- The package includes sortable candidate narrowing for large simple sortable sets, but that is an implementation optimization. Do not treat it as a virtualization API.

## Common Mistakes

- Do not use unstable ids such as array indexes when the order can change. Use persistent item ids.
- Do not treat `placementBoundary` as a targeting threshold. Targeting chooses the active drop target; placement chooses before or after relative to that target.
- Do not commit order in `onDragUpdate`. Use `onDrop` and `event.sortablePlacement`.
- Do not forget containers for empty lists. Without a registered drop container, an empty list has no sortable item target.
- Do not infer the destination container from `dropTargetId`. Use `sortablePlacement.containerId` when sortable placement is present.
- Do not rely on stale measurements after app-owned layout changes. Call `controller.remeasureDropTargets` after layout changes that should affect targeting, and call `controller.recomputeActiveDrag` when the active target should update immediately.
- Do not re-register unnecessarily in vanilla code. Re-rendering from data is fine for small examples, but large lists should preserve elements where possible.
