# mk-drag-and-drop

`mk-drag-and-drop` is a small, headless drag-and-drop toolkit. It gives your app the behavior layer: input handling, element registration, measurement, targeting, lifecycle callbacks, overlays, modifiers, and sortable placement data. It does not ship prebuilt components or own your application state.

```sh
pnpm add @mk-drag-and-drop/dom
pnpm add @mk-drag-and-drop/react
```

Use `@mk-drag-and-drop/dom` when you are working with plain DOM, custom elements, another framework, or your own adapter. Use `@mk-drag-and-drop/react` in React 19 apps when you want a provider and hooks over the same DOM runtime.

## What The Package Owns

- Pointer and keyboard drag input.
- Draggable, droppable, drop-container, sortable, and drag-handle registration.
- Drop-target measurement and active-target selection.
- Temporary sortable preview movement during an active drag.
- Optional drag overlays, including manual release animations.
- Movement modifiers such as axis locks and container bounds.
- Lifecycle events that describe what happened.
- Explicit remeasurement and active-drag recomputation APIs for app-owned layout changes.

## What Your App Owns

- Markup, styling, layout, and rendering.
- Application data, ordering, persistence, and final drop commits.
- The meaning of ids such as `draggableId`, `dropTargetId`, `group`, and `containerId`.
- Accessibility labels, instructions, domain-specific announcements, and interaction design.
- Any animation beyond the package-owned overlay host movement and sortable DOM preview.

The package reports drag results. Your app decides what those results mean.

## Mental Model

1. Create one drag scope with `createDragController` or `DragProvider`.
2. Register drag sources with `createDraggable`, `useDraggable`, or `useSortable`.
3. Register targets with `createDroppable`, `useDroppable`, `createDropContainer`, `useDropContainer`, or sortable items.
4. Configure targeting, constraints, modifiers, overlays, pointer input, keyboard input, and announcements when the defaults are not enough.
5. Read lifecycle callbacks such as `onDragStart`, `onDragUpdate`, `onDragEnd`, and `onDrop`.
6. Commit your own data on drop and render the final state through your app.

`onDrop` runs only for a successful drop on a valid target. `onDragEnd` runs whenever a drag ends, including cancellation or release with no target.

## Current API Notes

Older examples or docs may assume a smaller API surface. The current packages expose these public patterns:

- React overlays are configured with the `dragOverlay` prop on `DragProvider`. There is no public `DragOverlay` component export.
- DOM controllers expose `remeasureDropTargets()`, `remeasureOverlay()`, and `recomputeActiveDrag()`. React exposes the matching `useRemeasureDropTargets()`, `useRemeasureOverlay()`, and `useRecomputeActiveDrag()` hooks.
- `overlayRelease="manual"` calls the overlay renderer again with `phase: "released"` and a `removeOverlay` callback so your app can finish a release animation.
- Sortable items can be vertical or horizontal through `axis`, and `placementBoundary` tunes same-target reversal behavior.
- Sortable drops may include `event.sortablePlacement`. Plain droppable drops are id-only.
- Binding helpers and hooks register DOM elements, but they do not commit app data or expose sortable reorder helpers. Reordering remains app code.

## Examples

- [Basic drag](/projects/mk-drag-and-drop/examples/basic-drag): one draggable item, two droppable containers, an overlay, and a pointer-distance targeting constraint.
- [Sortable list](/projects/mk-drag-and-drop/examples/sortable-list): `useSortable`, sortable placement data, axis locking, overlay-center targeting, and manual overlay release.
- [Kanban](/projects/mk-drag-and-drop/examples/kanban): sortable columns, sortable cards, drop containers, composed refs, and a container-bound modifier.
- [Groups](/projects/mk-drag-and-drop/examples/groups): sortable parent rows, draggable child rows, generated insertion targets, custom constraints, and explicit remeasurement after expansion changes.
- [Tree](/projects/mk-drag-and-drop/examples/tree): app-owned tree projection, generated drop targets, custom targeting, and hierarchy updates from `dropTargetId`.

The examples intentionally keep data commits inside the app. They are useful as integration patterns, not as required data structures.
